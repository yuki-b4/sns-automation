## 3. Threads → note の一貫性評価

### テーマ一貫性: ◯（高い）
Threads の `[structure]` 系（「昼休みにちゃんと休んでいる人ほど…」「休日にちゃんと休めていると思っている人ほど…」）は、逆説型テーマと骨格が同じ。`[dialogue]` 系（「あと5分だけが30分」「早く帰りたいじゃなくて帰って何する」）は場面描写型と直結。**Threads 資産は note 側に十分活かせている**。

### threads_to_note_handoff の手書き運用フェーズでの実践度
- **◯ 達成例**: 旧仕様時代の 30 代エンジニア関連 1 記事（4 月末公開）で、Threads「早く帰りたいじゃなくて帰って何する」をそのまま冒頭引用ブロックで回収。 *(初版で具体タイトルを引用していたが該当ファイル名と一致しないため取り下げ。手書き運用に切り替わったあとも実践可能と判断)*
- **△ 不足例**: 共感最大化系記事はThreads の問いを冒頭300字以内で回収していないものが目立った。これが閲覧低迷の一因の可能性。
- **手書き運用への含意**: 運用者または Claude が note 本文を書く工程で、`config/note_writing_guide.json` の `engagement_design_rules.threads_to_note_handoff` を**毎回参照**できる導線（CLAUDE.md の本文作成節）が機能していれば、新仕様でも品質は維持できる。

## 4. 来週の推奨アクション（新アーキテクチャ前提）

### テーマ採択フェーズで意識すべきこと
`generate_note.py` は 3 テーマ提案で `target_brain` を `reptilian / mammalian / both` で必ず分散させるプロンプト設計。運用者が**3 案から 1 つを選ぶ際の指針**として：

1. **逆説型を含む reptilian 案を優先採択**: 旧スキーマ retrospective で「知的好奇心 / 逆説型」が最も効きそうだという仮説（弱いシグナル）が出ている。同等条件なら reptilian 案を選ぶ。
2. **mammalian 単独案は連続採択を避ける**: 共感最大化系の閲覧低迷観察（n=6, 平均16.3）に対応。所属・承認訴求のみで構成された case は、別日の reptilian 採択で挟む。
3. **both は重宝**: 1 件で両軸を刺激できる案は、コンセプト記事・有料note誘導の主軸に向く（例: 2026-05-08 提案「意志力で乗り切ってきた人が35歳で必ずぶつかる壁の正体」）。

### target_brain 分散の検証
今週分（2026-05-08〜05-11）の提案実績では、`reptilian + reptilian + mammalian` ／ `reptilian + mammalian + both` のような分散が確認できる（output/notes 確認可）。プロンプト側の分散指示は機能している。

### テーマ × target_brain の試行案
- 「知識武装と行動麻痺」× reptilian（損失回避・縄張り損失）—— 5/11 提案 1 を採択する場合
- 「妻の沈黙と関係性の喪失」× mammalian（所属・愛着崩壊）—— 既提案テーマの深掘り
- 「設計 vs 意志力」× both —— ミドルエンド／バックエンドへの導線づくり

## 5. 手書き note 本文ルールへの追加提案

> **重要**: `config/note_writing_guide.json` は現行 `generate_note.py` から参照されない。以下は **運用者・Claude が手書きで note 本文を書くときの参考資料に対する追記案**。`pattern_distribution.weights` や `free_mode_weekly_caps` の数値変更は提案しない（変更しても自動配分に影響しないため）。

### 追記候補（既存ルールと重複しない範囲）

1. **3 テーマ提案からの選定基準を明文化**: 「同等のフックの強さなら reptilian 案を優先」「mammalian 単独案を連続採択しない（直前1〜2日と target_brain を変える）」を `engagement_design_rules` に追加する案。
2. **共感→構造分析のハイブリッド設計**: 共感最大化型タイトルを採用する場合は、本文中盤で必ず「構造分析（なぜそのパターンが起きるかの解像度の高い説明）」ブロックを挟む。Before/After と失敗談の併用は単調化リスクが高い旨を `title_anti_patterns.phenomenon_description_warning` の関連注として追加する案。
3. **行動変容型の感情接続ステップ**: ステップ提示の前に「場面描写」ブロックを 1 つ挟むテンプレを `high_performance_patterns` に並ぶ新エントリとして追加する案（例: `action_with_emotional_anchor`）。「いいね 0 / 閲覧 31」観察（n=1）への対策仮説。
4. **冒頭 300 字以内の Threads 引用ブロック**: 既存 `engagement_design_rules.threads_to_note_handoff` に **「引用ブロック（`> `）を冒頭リード文の前に配置することを優先パターンとする」** を `implementation` の補足として追記する案。
