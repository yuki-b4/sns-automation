## 2. パターン別評価

### 🔵 知的好奇心（★★★★★）
- **強み**: 「忙しいは客観的事実ではなく認知の結果」のような**認知の枠組み転換**が冒頭で提示され、読者の知的フックを掴んでいる。逆説型 × 構造分析型 × 設計図型の組み合わせが、「読み進めれば構造が見える」期待を作っている。
- **改善点**: 投稿数が3本と少なく、勝ちパターンを十分活用できていない。逆説型が2回続いている点は型のバリエーション不足。
- **推定パフォーマンス**: いいね **高** / 閲覧 **高** / コメント 中 / 転換率 **高**

### 🟠 ファン化（★★★☆☆）
- **強み**: 「30代エンジニアの帰宅電車」のような**生活シーンの一人称解像度**が高く、ターゲット明示型の効果が出ている。閲覧30は健闘。
- **改善点**: いいね2.0は閲覧に対して低め（6.7%）。場面描写で共感を得た後の「アクション喚起」が弱い可能性。プロトコル型の出口が抽象的になっていないか要点検。
- **推定パフォーマンス**: いいね 中 / 閲覧 中 / コメント 低 / 転換率 中

### 🟢 信頼構築（★★★☆☆）
- **強み**: 「30名以上のクライアント」「松竹梅3つの打ち手」と数字で骨格を提示。読了動機が明確。
- **改善点**: いいね1.0は最低水準。数字インパクト型は信頼を担保するが「いいね」を生みにくい。**冒頭に感情のフック**を1段足すと改善余地あり。
- **推定パフォーマンス**: いいね 低 / 閲覧 中 / コメント 低 / 転換率 **高**（CV狙いの型として機能）

### 🔴 共感最大化（★★☆☆☆）
- **強み**: 「2年前、私は…」の自己開示は型として正攻法。失敗談型は心理的距離を縮める。
- **改善点**: 8本投稿で閲覧16.3は明確に低い。**冒頭が「あるある描写」で完結しがちで、構造的洞察への橋渡しが弱い**。Before/After型を謳いながら「After」の解像度が冒頭に出ていない記事が散見される疑い。**量を減らして質に振るべき**。
- **推定パフォーマンス**: いいね 中 / 閲覧 **低** / コメント 低 / 転換率 低

### 🟣 行動変容（★★★☆☆）
- **強み**: 「意志に頼らず」「3つの回復術」とベネフィット直球型の見出しが強い。閲覧31は単発として悪くない。
- **改善点**: いいね0は要警戒。ステップ型は実用的だが感情共鳴を生みにくい。問題提起と構造分析の比重を上げる余地あり。
- **推定パフォーマンス**: いいね 低 / 閲覧 中 / コメント 低 / 転換率 **高**

## 3. Threads → note の一貫性評価

### テーマ一貫性: ◎
Threads全10投稿が「家族時間／罪悪感／効率化の罠／脳の仕様」という共通テーマで、note 5パターンすべてと地続き。テーマ設計は極めて整合的。

### Threads → note ハンドオフ評価

| Threads投稿（問い） | 対応note冒頭 | 300字以内回収 |
|---|---|---|
| [dialogue]「効率上げたのに、帰れない」の正体 | 知的好奇心「なぜ忙しい人ほど忙しいまま」 | ✅ 冒頭で同じ問いを引用、明示的回収 |
| [structure] 振り返りで疲れが増えた | （直接対応note見当たらず） | ⚠️ 未回収 |
| [permission] 家族への申し訳なさ | 信頼構築「家族への罪悪感から解放される」 | △ 200字以降で接続、ややゆるい |
| [opinion] 珈琲屋の仕込み・導線 | （直接対応note見当たらず） | ⚠️ 未回収 |

**評価**: 知的好奇心パターンはthreads_to_note_handoffを優秀に実践（冒頭引用ブロックで問いを再提示）。一方、**[structure]系・[opinion]系のThreads投稿がnoteに着地していない**。Threadsで撒いた問いがnoteで回収されないとSNS→note誘導の信頼が損なわれる。

### 誘導設計: △
note内からThreadsへの逆導線、Threads投稿内からのnote誘導の明示が不足している印象。Threads投稿の最後に該当note URLを貼る運用ルール化を推奨。
