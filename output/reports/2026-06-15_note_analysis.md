## 4. Threads → note の一貫性評価

投稿済みnote記事が無いため、ハンドオフの実態評価は不可能。ただし**素材の噛み合わせ**は分析できる：

- Threads `[structure]` 「距離を縮めたいと逃げたいが同時に来る」 → 提案側「深まる手前で起動する自己破壊」「親密さで起きる窒息感」と直結
- Threads `[permission]` 「重いって言われた記憶、まだブレーキに使ってないですか」 → 提案側「重いと言われた言葉の残響」群と完全一致
- Threads `[opinion]` 「婚活セミナーの傾聴／相性診断の批評」 → **提案側に対応テーマが無い**（市場批評の角度が note 提案に降りていない）

**懸念**: Threads が出している「市場批評」「身体感覚」「permission（許可）」系の角度が、note 提案では「損失提示」「神経構造解説」に翻訳される過程で失われている。手書き本文フェーズで `threads_to_note_handoff` ルールを活かすには、**冒頭300字に Threads の問い文をそのまま引用ブロック化**できる素材選定が必要だが、現状の theme_label は「構造解説」寄りで Threads の声色が乗りにくい。

## 6. 手書きnote本文ルールへの追加提案

- **冒頭引用ブロックに Threads 原文をそのまま貼る運用を標準化**: 現状の theme_label が抽象化されすぎているため、Threads の生の問い文（例: `[permission]` の「重いって言われた記憶、まだブレーキに使ってないですか」）を冒頭に置き、本文で噛み砕く構成にすると一貫性が担保しやすい。

- **「市場批評→個人体験→構造解説」の3層トグル**: 提案側が構造解説に寄りがちなので、本文執筆時に「冒頭は市場批評or個人体験で入り、構造解説は中盤以降」と順序を固定すると、Threads の声色が冒頭で消えない。

- **読了後アクションを "気づき1つ持ち帰る" 型に統一**: 行動指示型（〇〇しよう）より、Threads `[structure]` 系の「この誤分類に気づいているか」のような **問い返し型クロージング** の方が、提案テーマ群（神経構造・内側の自爆）との相性が良い。

- **同一系統テーマを連続投稿しない自己ルール**: 採択シグナル分析の通り、提案側で「結婚意識で冷める」「重いの残響」が飽和。手書き側でも**同系統は2週間で1本まで**と運用者側で自制すると、読者側の既視感を防げる。
