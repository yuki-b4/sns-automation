# 個別相談スクリプト（45分・無料）

最終更新: 2026-05-12

> 本ドキュメントはセミナー出席者向け個別相談（45分・無料／通常¥50,000/h相当 ＝ 45分換算 ¥37,500相当の希少性）のオペレーション・スクリプトである。バックエンド「愛を深め続けるマインド構築講座（¥550,000）」へのクロージングを目的とする。
>
> ファネル全体設計は `docs/funnel_design.md` を参照。本スクリプトは §2.5 個別相談会の運用版。

---

## 1. 設計原則

1. **押し売りしない**：参加者が「自分の窮屈感の前兆が分かった」と言語化できる状態に到達していなければ、講座を提案しない（`docs/funnel_design.md` §2.5 設計ルール準拠）
2. **言語化セッションそのものを価値として提供する**：参加者が「自分のパターンが分かった」体験を持ち帰ることが、契約有無に関わらない約束。これが¥50,000/h相当の中身
3. **観察者ポジション**：クライアントの事例は「クライアントの〇〇さん」形式で語り、自分の体験と区別する（`docs/author_profile.md` 準拠）
4. **検討期間 48時間**：その場で即決させず、検討期間を提示。期間内連絡なしは自動クローズ
5. **テーマ一貫**：「親密になるほど窮屈になっていた関係を、関係を壊しそうな前兆に気づくことで、愛が深まるほど安心できる関係に変える ── 結婚の幸せに踏み込める」を軸に進める
6. **成約率目標 50%**（`funnel_design.md` 層4-c）：押し売りせずにこれを達成するには、ヒアリングと言語化セッションでの質が全て。クロージング段階で粘らない

---

## 2. タイムテーブル概観

| 時間 | セクション | 目的 |
|---|---|---|
| 0-5分 | オープニング | 信頼関係構築・本日のゴール共有・希少性提示 |
| 5-20分 | 現状ヒアリング | 恋愛経験／窮屈の瞬間／前兆の体感を聴く（15分） |
| 20-35分 | 言語化セッション | 一緒に前兆を文字にする（15分） |
| 35-40分 | カスタムロードマップ提示 | 「あなたが前兆に気づけると次の関係はこう変わる」 |
| 40-45分 | 講座概要・検討期間提示 | バックエンド契約への橋渡し |

---

## 3. 各セクションのスクリプト

### 3.1 オープニング（0-5分）

**話す内容**:

> 〇〇さん、本日は45分のお時間をいただきありがとうございます。
>
> 今日のゴールは2つあります。
> 1つ目は、〇〇さんが過去の恋愛で『親密になるほど窮屈になる』感覚を持っていた瞬間を、一緒に言語化すること。
> 2つ目は、これからの関係で『関係を壊しそうな前兆』に気づけるための、〇〇さん専用のマインド設計の方向性を見つけることです。
>
> 先日のセミナーで『愛を深め続ける関係性構築戦略』をお伝えしましたが、今日はそれを〇〇さんの具体的なケースに落とし込んでいきます。
>
> 進行としては、最初の15分で〇〇さんの過去の経験を聴かせてください。次の15分で一緒に前兆を言語化します。そのあと5分で〇〇さんの場合のロードマップをお伝えし、最後の5分で講座の概要をお伝えする予定です。
>
> なお、この45分間は通常¥50,000/h相当のコーチング枠ですが、今回はセミナーにご参加いただいた方限定で無料でお受けしています。
>
> 質問は最後の5分でお受けしますね。それでは始めましょう。

**チェックリスト**:
- [ ] ゴール2つを明示
- [ ] 進行構造を予告（15分×2＋5分×2）
- [ ] ¥50,000/h相当の希少性を提示
- [ ] 質問タイムを最後に集約（途中で逸脱させない）

### 3.2 現状ヒアリング（5-20分）

3つの軸で15分聴く。

#### 軸1: 恋愛経験のリスト化（約3分）

**質問**:
- 「これまでの恋愛経験、ざっくり何回くらいありますか？」
- 「その中で、最も印象に残っている関係を1つ教えていただけますか？」

**狙い**:
- 反復パターンを掴むための母数把握
- 後段のヒアリングで参照する具体ケースを1つ確定

#### 軸2: 親密になるほど窮屈になった瞬間（約6分）

**質問**:
- 「その関係の中で、〇〇さんが『窮屈だな』『冷めるな』『重いな』と感じた瞬間を覚えていますか？」
- 「それは関係のどの段階で起きましたか？（出会って数ヶ月／親密期／結婚を意識した瞬間 等）」
- 「その時、〇〇さんは何をしましたか？（距離を取った／別れた／我慢した／話し合った）」
- 「相手から最後に言われた言葉で、印象に残っているものはありますか？」

**狙い**:
- `config/strategy.json` `persona.description` の「優しく接するのが辛くなる／親密になるほど窮屈／結婚を意識した瞬間に冷める」のどれが当てはまるかを特定
- `pain_points` #1（別れ際の言葉）の残存度合いを把握
- 反復パターンの段階（どの瞬間で壊すか）を特定

#### 軸3: 関係を壊しそうな前兆の体感（約6分）

**質問**:
- 「振り返って、〇〇さんが関係を壊しそうになる『前兆』を3つ挙げるとしたら何ですか？」
- 「それは、思考？感情？行動？どれが先に来ますか？」
  - 思考の例: 「この人で本当にいいのか」「タイミングが合わなかった」「もっといい人がいるかも」
  - 感情の例: 急に冷める／重く感じる／焦り／窒息感
  - 行動の例: 返事が遅くなる／距離を取る／本音を隠す／会う頻度が落ちる
- 「その前兆が出た時、〇〇さん自身は何が起きていると思いますか？」

**狙い**:
- セミナーで提示した3つの前兆（A/B/C型）のうち、参加者がどれに近いかを把握
- 言語化セッション（次パート）の素材集め

**ヒアリング中の内的チェック（口に出さない）**:
- [ ] pain #1（別れ際の言葉のトラウマ）が出てきたか
- [ ] pain #2（自責⇔他責ループ）が出てきたか
- [ ] pain #5（結婚への踏み込み難）が出てきたか
- [ ] pain #7（一人で抱える孤独）が出てきたか
- [ ] 反復パターンの自覚があるか／否定的か

**ヒアリング作法**:
- 答えにくそうな質問は深追いしない（特に別れ際の言葉）
- 「正解」を求めず、その人の体感を聴く
- 沈黙を恐れず、思考の時間を与える
- 共感の相槌は入れるが、解釈は急がない
- メモは要点のみ。表情と声色に集中

### 3.3 言語化セッション（20-35分）

ペルソナと発信者で一緒に「あなたの場合の前兆パターン」を文字にする。本ファネル最大の価値提供パート。

#### ステップ1: 共通構造の提示（約3分）

**話し方の型**:

> お話を聴いて、〇〇さんの場合は『[共通構造の名前]』というパターンが見えてきました。
> これはセミナーでお伝えした[前兆A型／B型／C型]に近いです。
>
> 具体的には:
> - 〇〇さんは [きっかけ：相手が踏み込んでくる／結婚の話題／親密度の急上昇 等] が起きると
> - [思考：「この人で本当にいいのか」など] が浮かび
> - [感情：急な冷め／窒息感] が出てきて
> - [行動：距離を取る／本音を隠す] に出る
>
> このループの中で、最初に出てくる『前兆』は [X：思考か感情か行動か] だと思いますが、いかがでしょうか？

**狙い**:
- 参加者の語りを構造で返す（共感だけでなく観察も入れる）
- セミナーで聞いた『関係性構築戦略』の枠組みに、自分のケースが当てはまる体験を作る

#### ステップ2: 共同言語化（約8分）

ペルソナと一緒に、画面共有 or 手元のメモで以下を書き出す:
- 前兆の正体（一言で）
- それが出る具体的なシチュエーション
- 出てきた時に取りうるアクション（過去パターン／新パターン）

**質問**:
- 「この前兆を言葉にすると、〇〇さん自身の中で何が変わりそうですか？」
- 「これに気づけたら、次の関係でどうしたいですか？」
- 「過去のパターンで取ってきた[行動]の代わりに、何ができそうですか？」

**狙い**:
- 「分かった」が体験として残るよう、本人の言葉でアウトプットさせる
- ロードマップ提示に繋がる「次の選択肢」の素材を作る

#### ステップ3: 確認（約4分）

**核心の問い**:

> いま、〇〇さんの中で『自分の前兆パターンが分かった』感覚はありますか？

**応答パターンと対応**:

| 応答 | 解釈 | 次のアクション |
|---|---|---|
| 「はい、分かった／腹落ちした」 | 講座提案準備OK | 35-40分のカスタムロードマップへ進む |
| 「まだ分からない」 | 言語化不足 | ステップ2に戻り、別角度で言語化を試みる |
| 「分からないけど、何かが見えた」 | グラデーション | ロードマップ提示で確認しつつ進める／講座提案は弱めに |
| 「これは自分のことじゃない」 | パターン認識ミス | ヒアリングからやり直す or 講座提案せず別経路（ミドルエンドnote等）を案内 |

**判断基準**（講座を提案するか否か）:
- [ ] 参加者が自分の前兆を1〜3個、自分の言葉で言語化できた
- [ ] 「自分のことだ」と腹落ち感が表情・声に出ている
- [ ] 過去の経験との接続ができている

これらが満たされない場合は、講座提案ではなくミドルエンドnote（「理想の相手の見つけ方ガイド」シリーズ）や次回セミナー再参加を提案する。

### 3.4 カスタムロードマップ提示（35-40分）

**話す内容**:

> 〇〇さんの場合、今日言語化した前兆 [X] に気づけるようになると、次の関係でこう変わります。
>
> - **短期（最初の1ヶ月）**: [X] が出た瞬間に「あ、これは自分のパターンだ」と気づける状態を作ります
> - **中期（3ヶ月）**: 気づいた上で、過去とは違う選択を取れるようになります。具体的には [Y：本音を伝える／距離を詰めずに留まる など] 
> - **長期（6ヶ月〜1年）**: 関係を深め続けるマインドが土台として整い、相手と一緒に安心して結婚の幸せに踏み込めるようになります
>
> これを実現するために必要なのは、前兆に気づき続けて愛を深め続けるためのマインドです。そのマインドを構築する道筋を、ここからお伝えしますね。

**チェックリスト**:
- [ ] 短期／中期／長期の3スパンで提示
- [ ] AFTERの具体像を映像として描く（抽象論を避ける）
- [ ] 「マインド」をキーワードとして提示（講座名と整合）
- [ ] 次の5分（講座概要）への自然な橋渡し

### 3.5 講座概要・検討期間提示（40-45分）

**話す内容**:

> 〇〇さんの場合、このマインドを構築するために最適な道筋として、私が運営している『愛を深め続けるマインド構築講座』をご紹介します。
>
> [講座概要]
> - 期間: 〇ヶ月（運用者が確定）
> - 価格: ¥550,000
> - 内容:
>   - 個別のマインド設計セッション
>   - 関係性構築戦略の実践フィードバック
>   - 〇〇さんの場合のカスタム前兆対処設計
>   - 困った時のLINE個別サポート
> - ゴール: 〇〇さんが、ご自身の前兆に気づき続けながら、愛を深め続けられるマインドを完成させる
>
> 今、即決していただく必要はありません。検討期間として48時間をお取りしますので、その間にもしご質問があれば、LINEでお気軽にお送りください。
>
> 48時間以内にご返答がない場合は、今回のご案内は一旦クローズとさせていただきますが、それで何の問題もありません。〇〇さんが今日の言語化で得たものは、講座を取らなくても活きるはずです。
>
> 何か、この時点でご質問はありますか？

**押し売りNG ルール**:
- 「今日決めてください」と圧をかけない
- 価格を理由に引かれそうな場合、無理に説得しない
- 「分かりにくい」「決められない」と言われたら、「今は決めない」を選択肢として提示する

## 4. 質疑応答対応

### 4.1 よくある質問と回答方針

| 質問 | 回答方針 |
|---|---|
| 「価格が高い」 | 「他の方法でも改善は可能。講座は最短ルートの1つ。¥550Kは6ヶ月のマインド構築投資として位置づけている」 |
| 「自分にできるか不安」 | 「皆さん最初は同じ感覚から始まる」を伝え、不安を否定しない。具体的なクライアント事例（観察者立場で）を1つ紹介 |
| 「すぐに結果出ますか」 | 「マインドは時間がかかる。3ヶ月後から手応えが出る人が多い／即効性を求めるならノウハウ系の方が向いている」 |
| 「もう少し考えたい」 | 「48時間お取りします。LINEで質問可」と返す。期間の理由（決断疲れを避けるため設定）も短く説明 |
| 「次の交際相手ができてから受けたい」 | 「相手がいない時のほうが内側設計に集中できる／関係が始まってから内側を変えるのは難易度が上がる」を伝える |
| 「分割払いはできますか」 | 運用者の決済プラットフォーム設定次第（事前確認）。可能なら案内 |
| 「返金保証はありますか」 | 運用者の規約次第（事前確認） |

### 4.2 禁止フレーズ

- 「今だけ」「特別価格」（焦らせる表現）
- 「絶対」「必ず」「100%」（結果を保証する表現）
- 「他の人は皆」（同調圧力）
- 「あなただけ特別」（特別感の演出）
- 「これを逃すと次はない」（FOMO 演出）
- 「クライアントの〇〇さんも[具体的成功談]」を直接的に結果保証として使う（観察者ポジションを保つ）

## 5. 検討期間中のLINE対応

参加者から検討期間中に質問がきた場合の対応方針:

| 質問タイプ | 対応 |
|---|---|
| 内容の追加質問 | 1回まで詳細に回答。2回目以降は「個別に話した方が早いので30分の追加相談を設けますか？」と提案 |
| 価格の交渉 | 「価格はファネル全体の設計上、変動できません」と一貫して返す |
| 期間延長依頼 | 24時間まで延長可・それ以上は再案内サイクル（次のセミナー後の相談）に回す |
| 申込意思表示 | 決済リンクと契約書を即送付 |
| 「他のサービスと比較中」 | 比較対象の特徴を1度だけ説明。具体的な他社批判はしない |

### 48時間経過時の処理

- **申込なし・連絡なし**: 自動クローズ。追加メッセージは送らない（押し売り回避）
- 「次回機会があれば」と来た: 「次の機会はLINE配信でお知らせします」と返し、`status='cooling'` で `ファネル登録ログ` に記録

## 6. 失注時の対応

講座契約に至らなかった参加者への対応:

1. **離脱させない**: LINEブロックされない範囲で配信継続
2. **再アプローチのタイミング**: 3〜6ヶ月後、新しいセミナー案内 or ミドルエンドnote（「理想の相手の見つけ方ガイド」シリーズ）案内
3. **記録**: `ファネル登録ログ` の `status` を以下のいずれかに更新
   - `lost`: 明確に「受けない」と回答した
   - `cooling`: 「いつか機会があれば」など曖昧な状態

### 失注理由の分類記録

`ファネル登録ログ` の `notes` 列に以下のラベルで簡潔に記録（後段の改善分析用）:

| ラベル | 意味 |
|---|---|
| `price` | 価格がネックで離脱 |
| `timing` | タイミングが合わない（仕事繁忙等） |
| `partner_needed` | 「相手ができてから」と希望 |
| `unclear_value` | 講座価値が腹落ちしなかった |
| `wants_quick` | 即効性を求めていた |
| `other` | その他 |

## 7. ロール別チェックリスト

### 7.1 開始前（24時間前まで）

- [ ] 参加者の `ファネル登録ログ` を確認（セミナー出席日／LINE登録日／申込日時）
- [ ] セミナー後フォームの回答内容を読み込み、ヒアリングの仮説を立てる
- [ ] Zoomリンク／カレンダー招待が参加者に届いているか確認
- [ ] 講座決済リンク（Stripe等）が事前準備されているか確認

### 7.2 開始直前（5分前）

- [ ] Zoom起動・参加者の入室確認
- [ ] スクリプト（本ドキュメント）を画面外に開いておく
- [ ] 録画はしない（参加者のリラックスを保つため）

### 7.3 終了直後

- [ ] `ファネル登録ログ` の `consultation_implemented_at` に実施日時を記録
- [ ] `consultation_status` を `done` に更新
- [ ] 判断結果に応じて `status` を更新（contracted / cooling / lost のいずれか）
- [ ] 検討中の場合は48時間タイマーを記録（手動カレンダー）

### 7.4 48時間後

- [ ] 返答なしの場合、自動クローズ（`status='cooling'` または `lost`）
- [ ] 失注理由がはっきりしている場合は `notes` にラベル記入
