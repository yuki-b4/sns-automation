# 無料webセミナー（90分）スライド構成

最終更新: 2026-05-12

> 本ドキュメントはセミナー出席者向け90分セミナーのスライド構成・話す内容・運用設計の運用版。LINE登録者を理解段階から納得段階へ進める設計で、個別相談会（45分・無料／通常¥50,000/h相当）への申込導線を含む。
>
> ファネル全体設計は `docs/funnel_design.md` §2.4 を参照。個別相談スクリプトは `docs/consultation_script.md` を参照。

## 1. 概要

| 項目 | 内容 |
|---|---|
| 対象 | 公式LINE登録者（7日配信を完了した参加者）／LINE登録から最大3週間以内に出席 |
| 時間 | 90分（リアルタイム双方向） |
| 形式 | Zoom Pro 等のオンラインセミナー／**録画なし**（出席インセンティブ保持） |
| 開催頻度 | 月1〜2回 |
| 想定参加者数 | 立ち上げ初期 5〜20名 |
| ゴール | 1. 親密になるほど窮屈になる構造の理解／2. 3つの前兆の認識／3. 『関係性構築戦略』の体系受け取り／4. 個別相談会への申込 |
| 主要KPI | 出席→個別相談申込率 50%（`funnel_design.md` 層4-a 目標） |
| スライド枚数 | **20枚以内** |

## 2. 設計原則

### 2.1 オリジナルメソッド感の演出
『**愛を深め続ける関係性構築戦略**』を発信者オリジナルのメソッド名として一貫した語彙で使用する。LINE配信／個別相談／バックエンド講座すべてで同じ呼称。

### 2.2 「関係を壊しそうな3つの前兆」のネーミング規約
テクニック名ではなく**内側構造の名前**に揃える（`funnel_design.md` §2.4 設計ルール準拠）。本稿の暫定案は §4.3 に記載。運用者と最終確定する。

### 2.3 個別相談会の希少性訴求
最終スライドで「申込フォームURLは本日中限定／今月の枠は◯名まで」のような形で集約。セミナー本編中は直接訴求しない。

### 2.4 個別相談スクリプト 11原則の前段仕込み
`docs/consultation_script.md` §4.3 の「セッション前に仕込んでおくべき事項」を、本セミナーで重点的に仕込む:
- 自己投資の必要性（本物の情報は無料では手に入らない）
- 「学ぶ」では届かない領域があること
- 発信者自身が自己流で躓いた経験
- 「現在進行形で関係を深め続けている人と取り組む」価値

これらが弱いと、個別相談時の§3.6（自己流NGの根拠）で初めて聞く話になり説得力が落ちる。

### 2.5 インタラクティブ性
チャット投票・チャット質問を3〜4回挟む。録画なし設計と合わせて「この場でしか体験できない」感を作る。

### 2.6 観察者ポジション
クライアント事例は「クライアントの〇〇さん」形式で語る（`docs/author_profile.md` 準拠）。

### 2.7 テーマ一貫
「親密になるほど窮屈になっていた関係を、関係を壊しそうな前兆に気づくことで、愛が深まるほど安心できる関係に変える ── 結婚の幸せに踏み込める」を軸に進める。

### 2.8 「直接バックエンド訴求しない」原則
バックエンド講座（¥550,000）はセミナー内では訴求しない。個別相談会への申込が本セミナーの唯一のCTA（`funnel_design.md` `funnel.backend_path` 準拠）。

## 3. 全体タイムテーブル

## 4. スライド詳細（全20枚）

### Section A: オープニング（0-10分／スライド1-3）

#### スライド 1：タイトル

**レイアウト**: 中央配置・大きめタイトル＋小さく副題

**内容**:
```
親密になるほど窮屈になっていた関係を、
愛が深まるほど安心できる関係に変える

── 結婚の幸せに踏み込める自分へ

愛を深め続ける『関係性構築戦略』90分セミナー
```

**話す内容（0-1分）**:
> 〇〇です。今日はこの90分、お時間をいただきありがとうございます。今日のセミナーのタイトルは「親密になるほど窮屈になっていた関係を、愛が深まるほど安心できる関係に変える」です。

---

#### スライド 2：自己紹介

**レイアウト**: 左に顔写真＋右に経歴箇条書き

**内容**:
- 経歴：珈琲屋オーナー → エンジニア → エンジニア兼コーチ
- 結婚：している
- ポジション：コーチとして多数のクライアントの恋愛・人間関係を観察してきた立場
- 差別化：テクニックやノウハウに頼らず、自分が苦しくなる内側のパターンを言語化して書き換えるマインド設計型メソッド

**話す内容（1-3分）**:
> 簡単に自己紹介します。私は珈琲屋のオーナーをやっていた時期があり、その後エンジニアになり、今はエンジニアの仕事と並行してコーチをしています。
>
> 今日のテーマである「恋愛・関係性」については、コーチとして多数のクライアントの方の悩みを観察してきた立場でお話しします。私自身も妻と結婚していますが、結婚生活も「ありのままで愛し愛される関係を深め続ける」というテーマで、現在進行形で取り組んでいます。
>
> 大切なのは、今日お伝えする内容は「私の体験」だけでなく「複数のクライアントの方々を観察した結果見えてきた構造」であるということです。観察者の立場から、〇〇さんがご自身のパターンを発見する手がかりをお渡しします。

**仕込みポイント**:
- 「現在進行形で取り組んでいる」を強調（`consultation_script.md` 原則1.8「現在進行形で関係を深め続けている人と取り組む」の前提）
- 観察者ポジションを明示（`docs/author_profile.md` 準拠）

---

#### スライド 3：本日のゴール ＋ 参加者への問いかけ

**レイアウト**: 上半分にゴール、下半分にチャット問いかけ

**内容**:
```
[本日のゴール]
1. なぜ「親密になるほど窮屈になる」のかの構造を理解する
2. ご自身がやりがちな「関係を壊しそうな3つの前兆」を発見する
3. 『関係性構築戦略』を体系として持ち帰る

[チャット投票・1分]
質問：あなたはこれまでの恋愛で、
親密になるほど窮屈になった経験がありますか？

A. よくある
B. たまにある
C. あまりない
```

**話す内容（3-7分）**:
> 今日のセミナーで持ち帰っていただきたいゴールは3つです。
>
> 1つ目：「親密になるほど窮屈になる」というのが偶然ではなく、内側の構造から生まれるものだと理解すること。
> 2つ目：ご自身が無意識にやってしまっている「関係を壊しそうな3つの前兆」のうち、どれが自分に当てはまるかを発見すること。
> 3つ目：私が運営している『関係性構築戦略』というメソッドの全体像を、体系として持ち帰っていただくこと。
>
> ここで1つ、チャットで答えていただきたい質問があります。「あなたはこれまでの恋愛で、親密になるほど窮屈になった経験がありますか？」A/B/Cでチャットに入れてみてください。

**運用**:
- チャット参加率を確認（インタラクティブ性の最初の入口）
- 3〜5分の余裕を持ち、回答を見ながら「Aが多いですね」等の応答を入れる

---

#### スライド 4（スライド3の続き／必要に応じて）：今日の進行

**レイアウト**: タイムライン形式

**内容**:
```
[今日の進行]
10-30分：親密になるほど窮屈になる構造
30-60分：関係を壊しそうな3つの前兆 ＋ 関係性構築戦略
60-75分：私自身の脱却プロセス
75-85分：Q&A
85-90分：今後のステップのご案内

質問はチャットでいつでも投稿可能。Q&Aで取り上げます。
```

**話す内容（7-10分）**:
> 進行についてです。最初の20分で「なぜ親密になるほど窮屈になるのか」の構造を、次の30分で「3つの前兆と関係性構築戦略」を、その次の15分で私自身の脱却プロセスをお話しします。そのあと10分のQ&Aと、最後の5分で今後のステップのご案内です。
>
> 質問はチャットにいつでも投稿してください。Q&Aで取り上げます。
>
> なお、本日のセミナーは**録画していません**。これは私の経営方針ですが、リアルタイムで一緒に考える時間にしたいので、録画ではなく今この場に集中していただきたいと考えています。

**仕込みポイント**:
- 「録画なし」を明示し、希少性を作る

> 注：スライド数 20 枚以内の制約のため、スライド3 と 4 はレイアウト次第で1枚に統合可能。

### Section B: 親密になるほど窮屈になる構造（10-30分／スライド5-8）

#### スライド 5：脳科学的メカニズム ──「親密になるほど窮屈」が起きる理由

**レイアウト**: 中央に構造図（脳の機能図 or 簡略化したシステム図）

**内容**:
- 親密度が上昇するとき、脳のどの領域がどう働いているか
- 「結婚を意識した瞬間に冷める」現象の生理学的説明
- 自律神経・扁桃体・前頭前皮質 等の概念を簡潔に1つだけ提示

**話す内容（10-15分）**:
> まずは「なぜ親密になるほど窮屈になるのか」の構造からお話しします。
>
> 結論から言うと、これは性格の問題でも、相手選びの問題でもなく、**脳の自然な反応**として起きています。
>
> 親密度が上がる、つまり相手が自分の心理的なテリトリーに入ってくるとき、脳内では[脳科学的根拠1つ：扁桃体の反応／自律神経の切り替わり 等]が起きます。これが「窮屈感」として体感されます。
>
> 重要なのは、これは**反応であって、人格的な欠陥ではない**ということです。

**仕込みポイント**:
- ペルソナの自尊心を保つ（「あなたの性格の問題ではない」）
- 脳科学根拠を1つだけ提示（出典は明記しない・メカニズム説明として扱う）

---

#### スライド 6：自責⇔他責ループ図

**レイアウト**: 中央にループ図（自責→限界→他責→曖昧化→次回も同じ）

**内容**:
```
[関係が壊れたあと、内側で起きるループ]

「自分のせいだ」（自責）
       ↓ 耐えられなくなる
「タイミングが合わなかった」（他責・外的要因化）
       ↓ 一時的に落ち着く
[次の関係でも同じパターンを反復]
       ↓
「自分のせいだ」（自責）……
```

**話す内容（15-22分）**:
> 関係が壊れたあと、皆さんの内側ではこんなループが起きていませんか？
>
> 最初は「自分が悪かった」と自責します。優しい人ほどそう感じます。でも、その自責が一定期間続くと、自分が持たなくなるんですね。
>
> そこで、無意識に「タイミングが合わなかった」「相手の問題もあった」と外的要因に転嫁し始めます。これで一時的に楽になります。
>
> ただ、このループの問題は、**「次にどうすればよいか」にたどり着かない**ことです。自責と他責の間を行き来するだけで、自分のパターン自体は手付かずのまま、次の関係に入ってしまう。
>
> その結果、何が起きるかというと、**同じパターンを別の相手と繰り返す**わけです。

**仕込みポイント**:
- `config/strategy.json` `pain_points` #2 を直接言語化
- 「次にどうすればよいか分からない」という自覚を喚起

#### スライド 7：なぜ「学んでも」解けないのか

**レイアウト**: 左に「学習で得たもの」、右に「変わるために必要なもの」の対比

**内容**:
```
[心理学・脳科学を学んでも解けない理由]

学習で得たもの             変わるために必要なもの
─────────────             ─────────────────
・知識                   ・自分の内側を観察する力
・他人を分析する力       ・自分のパターンに気づく力
・「型」の理解           ・気づいた瞬間に選び直す力

[知識]と[気づき・選択]は、まったく別の能力
```

**話す内容（22-26分）**:
> ここで1つ、皆さんに聞いてみたいことがあります。
>
> このセミナーに参加されている皆さんは、おそらく心理学や脳科学の本を何冊か読んでこられた方も多いと思います。回避型／不安型といった愛着スタイルの概念をご存じの方もいるかもしれません。
>
> でも、その知識を**自分自身**に応用するのは、なぜか難しい。他人を分析するときには使えるけれど、自分が窮屈になった瞬間に「これは私のパターンだ」とは気づけない。
>
> これはとても自然なことで、「学ぶ」と「変わる」は脳の使い方がまったく違うからです。
>
> 学ぶには知識のインプットで足ります。でも、変わるためには、**自分の内側を観察できる状態を作る**必要があります。これは本を100冊読んでも到達しにくい領域です。

**仕込みポイント**（`consultation_script.md` §4.3 対応）:
- 「学ぶ→変わる」の意識転換（Principle 9 の前段）
- 自己投資の必要性の前提（Principle 10）

#### スライド 8：テクニック型ノウハウの限界

**レイアウト**: 中央に「テクニック」と「マインド設計」の対比図

**内容**:
```
[よくあるノウハウ]                 [本セミナーでお伝えするもの]
──────────────                  ────────────────
モテるための〇〇                 自分のパターンを観察する力
LINE返信術                       気づいた瞬間に選び直す力
心理学テクニック                 関係を深め続けるマインド

→ 相手を変える発想               → 自分が変わる発想
→ 短期的に効く                  → 一度習得すると一生使える
```

**話す内容（26-30分）**:
> 世の中には「モテるための〇〇」「相手の心を掴むLINE術」など、テクニック型のノウハウがたくさんあります。それで一時的にうまくいく方もいます。
>
> でも、本セミナーにご参加いただいている方の多くは、おそらくそれでは解けない壁にぶつかった経験があるはずです。テクニックで相手を引き寄せられても、親密になるほど窮屈になるパターンは、自分の内側にあるからです。
>
> ここから30分でお伝えするのは、テクニックではなく、**自分の内側のパターンを観察し、関係を深め続けるマインド**そのものです。これは一度習得すると一生使える土台になります。

**仕込みポイント**:
- 「テクニックでは解けない」の自覚を強化（pain #4）
- 後段の『関係性構築戦略』への期待醸成

### Section C: 3つの前兆 ＋ 『関係性構築戦略』（30-60分／スライド9-15）

#### スライド 9：「関係を壊しそうな3つの前兆」概要

**レイアウト**: 中央に3つの前兆の名称を並べる（A/B/C）

**内容**:
```
[関係を壊しそうな3つの前兆]

前兆A：親密反転型
　近づくほど、離れたくなる

前兆B：結婚スパイク型
　結婚が見えた瞬間、急に冷める

前兆C：優しさ消耗型
　大切にするほど、自分が疲弊する

これから1つずつ、構造・事例・気づけたときに何が変わるかをお伝えします
```

**話す内容（30-32分）**:
> ここからメインコンテンツです。私がコーチとして観察してきた中で、「関係を壊しそうな3つの前兆」が見えてきました。前兆A・B・Cと呼んでいます。
>
> 前兆A「親密反転型」：近づくほど離れたくなる感覚。
> 前兆B「結婚スパイク型」：結婚が見えた瞬間に冷める感覚。
> 前兆C「優しさ消耗型」：大切にするほど自分が疲弊する感覚。
>
> 多くの方は3つのうち1〜2つが主軸になっています。1つずつ、構造とクライアントの事例、そして気づけたときに何が変わるかをお伝えします。

**運用注**:
- 前兆A/B/C のネーミングは**暫定案**。運用者と最終確定する。条件：内側構造の名前であり、テクニック名ではない（`funnel_design.md` §2.4 設計ルール準拠）

#### スライド 10：前兆A「親密反転型」

**レイアウト**: 左に構造図（きっかけ→思考→感情→行動の流れ）、右にクライアント事例

**内容**:
```
[前兆A：親密反転型 ── 近づくほど、離れたくなる]

[構造]
きっかけ：相手が踏み込んでくる
   ↓
思考　 ：「窮屈だ」「監視されている」
   ↓
感情　 ：身体が縮む感覚／呼吸が浅くなる
   ↓
行動　 ：返事が遅くなる／距離を取り始める

[クライアントの事例]
クライアントの〇〇さん（30代男性）の場合：
「相手から連絡頻度を上げてほしい」と
言われた瞬間に冷め始めた

[気づけたときに何が変わるか]
→ 「あ、また親密反転が起きてる」と
　 ラベリングできるようになる
→ 距離を取る前に立ち止まれる
```

**話す内容（32-40分）**:
> 前兆A「親密反転型」は、相手が一歩踏み込んでくる場面で起きやすいパターンです。
>
> 構造としては、相手が「もっと一緒にいたい」「連絡をもう少し取り合いたい」など、関係を一段深めようとした瞬間に、自分の側で「窮屈だ」「監視されている」といった思考が浮かびます。同時に身体が縮むような感覚や呼吸が浅くなる感覚があり、行動として返事が遅くなったり、距離を取り始めたりします。
>
> クライアントの〇〇さん、30代男性の方の例でお話しします。〇〇さんは交際相手から「連絡をもう少し増やしてほしい」と言われた瞬間に、急に冷めるパターンを繰り返していました。〇〇さんと一緒にこのパターンを言語化したところ、「親密度が上がるリクエストが来ると、自分の中の何かが反転する」ことが分かりました。
>
> 気づけるようになると何が変わるかというと、「あ、また親密反転が起きてる」とリアルタイムでラベリングできるようになります。これだけで、過去とは違う選択を取る余地が生まれます。

各セミナー実施後、以下のフィードバックを記録:
- 参加者のチャット投票結果（3つの前兆の分布）
- Q&Aで出た質問のパターン
- 申込率の推移
- 申込者のセミナー中の反応パターン

これらを月次で振り返り、スライドの該当箇所を更新する。
