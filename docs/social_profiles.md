> **本ファイルは実際のプロンプト生成に利用されていません。** 各プラットフォーム（Threads / note）のアカウントプロフィールに手動で設定する内容のリファレンスです。

## アカウント名（Threads / note 共通）

```
トーサイ | 自然体を愛してくれる夫の作り方
```

## プロフィール文章（ディスクリプション｜Threads / note 共通）

```
夫から自然体でも愛される自分づくり / 周りに依存せずしなやかに生きる女性のマインドをデザイン / 既婚男性が伝える離婚や別居する前に知りたい男のアレコレ
```

## 設計の意図

### 全体コンセプト
既婚男性の視点から、**自然体を愛してくれる夫の作り方**を伝える。読者は結婚歴3〜15年の既婚女性。発信者（既婚男性）は「夫側の本音を翻訳する当事者」であり、同時に「女性が自分の内側を整えるプロセスに伴走するコーチ」でもある。

### 役割分担
- **アカウント名**「自然体を愛してくれる夫の作り方」: “夫を変える”外向きのキャッチで手を止めさせる**入口フック**。手に取りやすさを担う
- **ディスクリプション**: 入口フックの中身を明かし、両輪を宣言する
  - 節1・2「夫から自然体でも愛される自分づくり」「周りに依存せずしなやかに生きる女性のマインドをデザイン」＝ **自分づくり（輪B）**。バックエンド「愛される自分を取り戻すパートナーシップ講座」と直結
  - 節3「既婚男性が伝える離婚や別居する前に知りたい男のアレコレ」＝ **夫翻訳（輪A）**。発信者が既婚男性であることの独自性

### 投稿の背骨（両輪の出し分け）
プロフの「夫の作り方」（入口）と中身の「自分づくり」（本質）の方向差は、投稿タイプで**明確に出し分ける**：
- **自分づくり（輪B）**: `structure` / `permission` — 読者自身の内側のパターン（尽くすほど満たされないループ）を言語化・書き換える
- **夫翻訳（輪A）**: `opinion` / `personal` — 既婚男性だからこそ語れる“夫側の本音・機微・行動の理由”を翻訳する
- **橋渡し**: `dialogue` — 経験開示＋問いかけで両輪をつなぐ

（詳細は `config/strategy.json` の `post_types.*.description` 冒頭の `【自分づくり】/【夫翻訳】/【橋渡し】` マーカー参照）
