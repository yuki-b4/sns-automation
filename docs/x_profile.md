# X（Twitter）プロフィール・固定ポスト

> **本ファイルは実際のプロンプト生成に利用されていません。** Xアカウントのプロフィール／固定ポスト（X Articles）／補助プロンプトを手動で設定するためのリファレンスである。`docs/author_profile.md` と同じ位置づけ。

X は Threads と並列の販売動線として運用する。バックエンド（戦略設計講座）／ミドルエンド（有料note ¥1,980）への遷移を担う。プロフィールは「名前欄＝目指したい人物像」「bio＝そこに至るメカニズム」、固定ポスト（X Articles）＝「権威性の証明＋note 誘導」の三層で構成する。

---

## プロフィール名

```
トーサイ｜残業ありき→定時で帰るハイパフォーマー
```
（21字）

### 設計意図

- **BEFORE→AFTER 型ハイブリッド**：`config/strategy.json` の persona pain_points #1（成果は出ているのに残業から抜け出せない構造的ジレンマ）を BEFORE「残業ありき」で表現
- **AFTER「定時で帰るハイパフォーマー」**：逆説のフックでタイムライン上で目を引かせる。「定時で帰る」と「ハイパフォーマー」の組み合わせが矛盾語法として印象に残る。成果ニュアンスは"ハイパフォーマー"が暗黙に担う
- **「設計」を名前欄から外した**：抽象すぎる／業界被りが多い／3秒勝負の場では意味解釈の負荷を下げる必要がある。bio 側で「行動設計」「両立設計」として残す
- **「家族時間」が名前欄から消える点**：あえて bio で完全回収する分業設計。名前欄を短く保つことを優先

## bio

```
フォローすると「残業しないと結果が出ない」から抜け出せる｜珈琲屋→エンジニア→マインドコーチ｜心理学・脳科学ベースの行動設計で、家族時間と仕事の成果を両立｜両立設計の具体ステップはnoteに書いてます↓
```
（約95字 / 160字制限内）

### 設計意図

| 要素 | 役割 |
|---|---|
| 「フォローすると〜から抜け出せる」 | フォロー CTA を冒頭に置く構造／セリフ型 BEFORE での自分事化／口語トーンで親密感 |
| 「」内ペルソナセリフ | pain_points #1 の口語訳。ペルソナの自尊心を保ったまま課題提示 |
| 珈琲屋→エンジニア→マインドコーチ | 異色キャリア＝`positioning.differentiation` の中核。X は権威性が刺さる媒体なので必須 |
| 心理学・脳科学ベースの行動設計 | 差別化軸＋メカニズム提示 |
| 家族時間と仕事の成果を両立 | 約束（名前欄から消えた両立軸をここで回収） |
| 両立設計の具体ステップは note に書いてます↓ | ミドルエンド（有料note ¥1,980）への口語誘導 |

### システムとしての整合
- **名前欄**＝目指したい人物像（「定時で帰るハイパフォーマー」）
- **bio**＝そこに至るメカニズム（行動設計／心理学・脳科学）
- 3秒読みで「停止 → 自分事化 → 権威確認 → 約束 → CTA」が完結する

### 自称ハイパフォーマー問題の吸収
名前欄の「ハイパフォーマー」は自称色がやや出るが、bio 冒頭「**フォローすると〜抜け出せる**」で読者主語に切り替わるため、「私もそうなれるのか」という読者側の解釈に変換される構造で吸収している。

## 固定ポスト（X Articles）

X Articles は Premium 以上で利用可能な長文機能。タイムラインのスレッドと違い、保存・後で見る前提の長文資産として機能する。

### 伸ばし方の要点（運用方針の根拠）

| # | 要素 | 効き方 |
|---|---|---|
| 1 | タイトル | 具体ベネフィット＋数字／固有名詞＋"少数発信"ニュアンス。検索流入も担う |
| 2 | 冒頭3行 | 既存常識の否定→"だから違う"の再提示。スクロール停止装置 |
| 3 | H2/H3 スキャン構造 | 飛ばし読みでも価値が伝わる骨格。保存動機を作る |
| 4 | 新規性ハードル | 既知ハック羅列は無風。まだ少数しか発信していないノウハウだけが拡散される |
| 5 | 具体プロンプト・手順 | 抽象論ではなく今日試せる粒度。引用RP対象になりやすい |
| 6 | 保存・引用RP動機 | 章の境目に"核となる一文"を置くと引用される |
| 7 | 末尾 CTA 分業 | フォロー（継続接点）／いいね・RP（拡散）／note 誘導（収益）の3層を分けて配置 |

### 本文（X Articles エディタ貼り付け用プレーンテキスト）

X Articles エディタはマークダウン記法を貼り付けても自動変換されない（行頭で `# ` を打つとその場で見出しになるショートカットはあるが、貼り付けは別）。下記をそのまま貼り、後述の「書式適用メモ」に従ってエディタ側で見出し・太字・箇条書きを適用する。

```
AIで残業が減っていく時短仕事設計術——誰も言ってないClaude/Gemini/ChatGPT"頭脳代行"3選

ChatGPTに議事録を要約させても、退社時刻は1分も早くならない。
Claudeにメールを書かせても、定時で帰れるようにはならない。

理由は単純で、世間で語られるAI活用の99%が「アウトプットを作らせるハック」だからである。
本当に残業を減らすのは、頭脳労働そのものをAIに肩代わりさせる設計だ。

30代エンジニア・PMが1日に下す判断は数百回を超える。夕方になると頭が回らないのは、体力ではなく判断疲れ（decision fatigue）が原因だ。ここを丸ごとAIに移譲できれば、残業は構造的に減る。

本記事では、Claude / Gemini / ChatGPT それぞれの強みを活かした"頭脳代行"の3つの活用法を紹介する。いずれも、よくあるプロンプト集には載っていない、コーチング現場で実際に組んでいるニッチな型である。


1. Claudeを"自分の判断基準のキャッシュ"として使う

汎用ハック：「これってどう判断すべき？」と単発で聞く
設計術：過去の判断ログを学習させ、"自分の代理判断者"を作る

エンジニア・PMの残業の正体は、コードを書く時間よりも「これでいいか迷う時間」だ。レビューコメントの返信、仕様の細部決定、優先度判断——1案件あたり数分の迷いが、1日で2〜3時間に膨れ上がる。

Claude Projectsに、自分の過去3ヶ月分の意思決定例を投入する。

例：
Aの場合は〇〇と判断した。理由は△△
Bの場合は✕✕を選んだ。なぜなら□□

これを10〜20件貼り、システムプロンプトに「この人の判断基準を再現して、迷う案件に答える」と指示する。

迷った案件を投げた瞬間、「あなたの基準ではAを選ぶはず。理由は〜」と返ってくる。
重要なのは、Claudeが正解を出すことではない。「自分の基準ならこう」と即座に再提示されることで、判断の8割が"確認"で終わる。

これは判断疲れの外部化である。Claudeを"判断のキャッシュメモリ"として使うイメージが近い。


2. Geminiで"退社時刻からの逆算タスク選定"を任せる

汎用ハック：Geminiにメールを下書きさせる
設計術：Google Workspace統合を活かし、"今日18時に帰るための1件目"だけを選ばせる

Geminiの最大の強みは、Gmail・Calendar・Drive とネイティブに繋がっていることだ。これを「タスク選定の文脈エンジン」として使う。

朝、Gemini Advancedにこう投げる。

今日のCalendarと未読Gmailを横断し、18時退社を達成するために最初の1件目に着手すべきタスクを1つだけ選んで。理由も添えて。

Geminiは予定の隙間と未処理の重要メールを照合し、「Aさんへの返信を9:30までに片付けないと午後の会議で同じ議題が再燃する」のように"残業を生む種"を先回りで指摘してくる。

ここでのコツは「優先度を10件並べてもらう」ではなく「1件だけ選ばせる」こと。脳は朝の最初の判断で1日の方向性が決まる。10件渡されても結局迷うが、1件なら即着手できる。

これは優先度判断そのものをAIに渡す設計である。


3. ChatGPTで"退勤儀式"を自動化する

汎用ハック：ChatGPTに業務報告のフォーマットを作らせる
設計術：仕事⇔家庭のスイッチを切る5分の対話を、毎日同じプロンプト1つで回す

ペルソナ最大のペインは「家に帰っても頭が仕事から離れないこと」である。これは脳科学的にはツァイガルニク効果——人は未完了タスクを記憶に保持し続ける——として知られる。閉じる手順がない限り、脳はタスクを離さない。

ここでChatGPTを「退勤コーチ」に仕立てる。カスタムGPTを組む必要はない。1つのプロンプトを使い回すだけでいい。

退社前、ChatGPTで新規チャットを開き、下記プロンプトを貼るだけ。
（プロンプト本文は後掲）

するとChatGPTが1問ずつ問いかけてくる。

1. 今日完了したことを5つ挙げてください
2. 頭に残っている未完了は？それぞれ明日のいつ手をつけますか？
3. 家に帰った後、頭に残りそうな仕事の残像は何か、1つだけ

答えるのに5分。終わったらチャットを閉じる。履歴を残す必要すらない。重要なのはAIに言うことではなく、口に出すことだからだ。

ChatGPTは何かを覚えている必要も、毎日のログを蓄積する必要もない。「毎日同じ問いを、同じ温度感で投げ返してくれる相手」として機能するだけでいい。タスク管理ツールへの依存はゼロだ。

これはタスク管理ではなく、"頭の中の仕事を物理的に閉じる手順"の自動化である。家に帰った後の家族との時間の質が、これだけで変わる。


3つに共通する"設計思想"

ここまでの3つに共通するのは、AIにアウトプットを作らせていない点だ。

Claude → 判断の代行
Gemini → 選定の代行
ChatGPT → スイッチの代行

すべて、人間の頭脳労働の中でも消耗の激しい工程だけを切り出して、AIに肩代わりさせている。残りの「考える」「決める」「やる」は人間が担う。

ChatGPTにブログ記事を書かせても残業は減らない。
ChatGPTに退勤前の頭の整理を任せると、今日から30分早く帰れる。
この差が、「AIで残業が減る／減らない」の分水嶺である。


ただし、これらも表面のハックに過ぎない

正直に書いておく。
ここまで紹介した3つは、効果はあるが根本解決ではない。

AI活用がうまくいくかどうかは、その手前にある「自分の働き方そのものをどう設計しているか」で決まるからだ。判断基準が曖昧なままClaudeに学習させても出力はブレる。退社時刻を本気で守る覚悟がない人がGeminiに逆算させても、結局残業する。

AIは、すでに設計された働き方を加速させるツールであって、設計のないところに設計を生み出すツールではない。

その「働き方の設計」をどう組むか——家族時間を取り戻すための具体ステップは、別途noteにまとめている。AIハック以前の本丸の話だ。

→ プロフィールのリンクからnoteへ


この記事が役に立ったら

リポスト：同じ悩みのエンジニア・PMに届けてほしい
いいね：書く励みになる
フォロー：こうしたニッチなAI×働き方設計の話を毎日流している

AIは表面のハックに過ぎない。
本質は、働き方の設計にある。
詳細はプロフィールのnoteから。
```

### 書式適用メモ（X Articles エディタ側で操作）

X Articles エディタはツールバー or `/` コマンドで書式を当てる Notion 型 UI。貼り付け後に下記を適用する。

| 行 | 適用書式 |
|---|---|
| 先頭の `AIで残業が減っていく時短仕事設計術——…` | 見出し1（H1） |
| `1. Claudeを"自分の判断基準のキャッシュ"として使う` | 見出し2（H2） |
| `2. Geminiで"退社時刻からの逆算タスク選定"を任せる` | 見出し2（H2） |
| `3. ChatGPTで"退勤儀式"を自動化する` | 見出し2（H2） |
| `3つに共通する"設計思想"` | 見出し2（H2） |
| `ただし、これらも表面のハックに過ぎない` | 見出し2（H2） |
| `この記事が役に立ったら` | 見出し2（H2） |
| 各セクション冒頭の `汎用ハック：…` `設計術：…` | 行頭の「汎用ハック」「設計術」をボールド |
| `例：` 以下の `Aの場合は〇〇…` `Bの場合は✕✕…` の2行 | 箇条書き |
| `今日のCalendarと未読Gmailを横断し、…` の段落 | 引用ブロック（プロンプト例として浮かせる） |
| 「3. ChatGPTで〜」内の `1. 今日完了したことを…`〜`3. 家に帰った後…` の3行 | 番号付きリスト |
| `Claude → 判断の代行` `Gemini → 選定の代行` `ChatGPT → スイッチの代行` の3行 | 箇条書き |
| `→ プロフィールのリンクからnoteへ` | 太字＋やや大きめ／可能なら note URL をリンク化 |
| `リポスト：…` `いいね：…` `フォロー：…` の3行 | 箇条書き |

### 設計方針

- タイトルで「誰も言ってない」「3選」「Claude/Gemini/ChatGPT」の3要素を入れて新規性を担保
- 冒頭で世間のAI活用の99%を否定し、独自フレーム「AIに**アウトプットではなく頭脳労働を肩代わりさせる**」を提示
- 3メソッドを **Claude=判断 / Gemini=選定 / ChatGPT=スイッチ** で機能分担し、AI3社の強みを別々に活かす（既存発信との差別化）
- 各メソッドに「汎用ハック vs 設計術」の対比を入れ、読者の既存知識を肯定的に上書き
- 末尾で「ここまでも表面のハックに過ぎない」と自分でハシゴを外し、本質=働き方の設計→note 誘導へ繋ぐ（自著否定からのCTAは信頼を上げ、note 遷移率を引き上げる定番技）

### 運用補足

- **公開タイミング**：火・水・木の朝7-9時 or 夜21-23時が記事系の伸びゾーン。月曜は週初タスクで読まれにくい
- **公開直後の引用ポスト**：記事URL＋核となる一文を抜粋した引用ポストを別途流すと、記事への流入が増える。例：「**判断疲れの外部化**こそAI活用の本丸である理由を書いた。多くの人がChatGPTに議事録要約させてる時点で間違えている」
- **固定ポスト化**：投稿後にプロフィールから「ピン留め」設定を実行
- **note 側との整合**：bio 末尾の note リンクを「働き方設計の総論的なもの」に都度入れ替えておくと離脱せずに購入率が上がる
- **画像挿入**：任意。入れるなら「3つに共通する"設計思想"」の上に Claude／Gemini／ChatGPT のロゴ並びの図を1枚置くとシェア時の OGP が映える

## 補助プロンプト：ChatGPT 退勤儀式

固定ポスト本文セクション3で言及している、退勤コーチ用のプロンプト本体。カスタムGPTを組まず、退社前に新規チャットを開いて貼るだけで動く。ChatGPT 無料アカウントでも動作。Claude / Gemini にもそのまま流用可能。

```
これから5分の「退勤コーチング」を始めてほしい。
あなたは対話相手であり、タスク管理者ではない。
私のタスクを覚えたり記録したりする必要はない。
以下の手順で、1ターンに1つだけ質問してください。

【手順】
1. まず「今日完了したことを5つ挙げてください」と聞く。
   回答が4つ以下なら「もう1つだけ思い出せますか？」と一度だけ促す。
   5つ揃ったら「お疲れさまでした」とだけ短く受け止める。

2. 次に「今、頭に残っている未完了タスクはありますか？」と聞く。
   私が挙げた各タスクに対し、1件ずつ
   「それは明日のいつ手をつけますか？」と問い返す。
   時刻が返ってきたら、それを受け止めるだけ。
   提案・段取りアドバイス・優先度の意見は出さない。

3. 最後に「家に帰った後、頭に残りそうな仕事の残像は何か、
   1つだけ言葉にしてください」と質問する。
   回答を受け止め、「明日また会いましょう」で対話を閉じる。

【守ること】
- 1ターンに1つの質問だけ
- 解決策・アドバイス・評価・励まし系の相槌は出さない
- 端的で落ち着いたトーン
- 質問を複数並べない

それでは、最初の質問からお願いします。
```

### 起動方法のおすすめ
- ChatGPT の Saved Prompts に保存
- macOS のテキスト置換で `;tk` などの短縮コードに登録
- Notion / メモアプリにピン留め
- ブラウザのブックマークレットに仕込む

これらいずれかで、退社前に1クリック→ペーストで起動できる。

## X 固有の「売れる理由」分析（参考）

Threads と比較した X の構造的特徴。バックエンド／ミドルエンド販売観点での運用判断の根拠。

| 観点 | X 固有の強み | 商品販売への効き方 |
|---|---|---|
| 流入 | X 内検索＋Google 検索の両方にインデックス | 「行動設計」「家族時間」「エンジニア 残業」等のキーワードで継続流入 |
| 拡散 | 引用RP/RTでフォロワー外に飛ぶ | 体系化系・自己開示系がバズで一気に bio 遷移へ |
| 動線 | ポスト→プロフ遷移率が高い／固定ポストがLP代わりになる | 固定ポストを有料note誘導、bioを講座導線に分担できる |
| ペルソナ密度 | 30代エンジニア・PM がまだ X メイン | ペルソナと媒体が完全一致 |
| 権威性の刺さり方 | 肩書き・経歴・実績のワンライナーで一目評価される文化 | 「珈琲屋→エンジニア→コーチ」の異色キャリアが最も効く媒体 |
| bio 制約 | 160字／名前50字（実質日本語25字程度） | 共感ではなくポジショニング1点突破が正解 |

→ Threads は「人柄・温度感」で買わせる場、X は「この人の話を聞く価値がある」を3秒で証明する場、と役割分担する。
