# Threads / X プロフィール 固定ポスト

> **本ファイルは実際のプロンプト生成に利用されていません。** Threads / X の固定ポスト（ピン留め）を運用者が手動で設定するためのリファレンス。`docs/author_profile.md` と同じ位置づけ。

固定ポストは「タイムラインで流れていく投稿の入口」とは別レイヤーの資産で、プロフィール訪問者が必ず最初に読む長文枠。役割は以下:

- **認知** → ペルソナ pain の名指し（「学んだ知識が恋愛で使えないIT エンジニアへ」）
- **共感** → 同じパターンに苦しんだ過去・別れ際の言葉・無意識ループの可視化
- **興味** → 「テクニックでは変わらない／無意識の自己犠牲パターンの書き換え」というオリジナル概念の提示
- **CTA** → 無料 note（プロフィールリンク）

## 固定ポスト

### タイトル

```
学んだ知識が恋愛で使えないITエンジニアへ
```

### 本文

```
恋愛で大事な心理学を読んでも、脳科学を学んでも、何冊本を積み上げても、いざ実践したら上手くいかなかった。

頭ではわかってる。
「ありのままでいい」
「自己開示しよう」
「Iメッセージで伝える」

理論はよくわかる。それなのに、関係が深まるほど自分が苦しくなって、最終的には自分の側から壊してしまう。

あの子に別れ際にかけられた
「重い」
「いい人なんだけど」
「察してほしかった」
という言葉が、いつも頭の片隅に残っている。

「次は同じ失敗をしない」と決めるのに、
気づいたらまた同じ場所で同じパターンを繰り返す。

あなたは、もうこれ以上知識は必要ないです。

なぜなら、あなたの大切にしたいという思いが、最終的に歪んでしまっているから。

実際に起きていることは、いつもこのパターン。

1. 相手を大切にしたいと思い始める
2. 相手の期待に応えようとする
3. 自分の本音や想いを後回しにし始める
4. 自分が苦しくなる
5. 冷める／距離を取る／壊してしまう

しかもこのパターンは、無意識に発動しています。

だから、本で読んで頭で理解しても、いざ実践になるといつも同じ結果になる。僕もそうだったからよくわかるけど、これはテクニックやノウハウでは変わらない。

「幸せな結婚生活を送りたいと思っているITエンジニア」のために、この「無意識の自己犠牲パターン」からどうやったら抜け出せるのかを発信していきます。

僕自身、心理学を読み漁っても恋愛が何も変わらなかった一人です。でも、自分の内面を一つずつ言語化して書き換えてきた結果、今は妻と良好な関係を継続できています。

元エンジニアの現役コーチとして多数のクライアントの内面と向き合ってきた経験が、この発信の土台です。

詳しく書いた無料noteは、プロフィール欄のリンクから。
```

## 設計意図

### 二フレーム構成（タイトル＝pain / 本文宣言＝aspiration）
- **タイトル**：「学んだ知識が恋愛で使えないITエンジニアへ」← pain 主導で名指し、クリック・読了動機を作る
- **本文中盤の宣言**：「幸せな結婚生活を送りたいと思っているITエンジニア」のために発信、← aspiration 主導で記憶に残す
- pain でクリックさせ、aspiration で「自分が辿り着きたい場所はここだ」と上書きする 2 段構え

### 自称は「僕」で統一
本文全体を通して一人称は「僕」。`docs/author_profile.md` の事実情報（夫婦／コーチ）と整合。

### 「あなたは、もうこれ以上知識は必要ないです」
読者がこれまで積み上げてきた知識追求行動を一度否定し、内面のパターン書き換えへ意識を反転させる中核フック。共感ブロック（pain 提示）と numbered list（メカニズム提示）の境目に置く。

### 5 ステップの numbered list
「無意識の自己犠牲パターン」を可視化。

1. 相手を大切にしたいと思い始める
2. 相手の期待に応えようとする
3. 自分の本音や想いを後回しにし始める
4. 自分が苦しくなる
5. 冷める／距離を取る／壊してしまう

ステップ 3 は「自分を消す」のような強い語ではなく「本音や想いを後回しにし始める」とソフトに具体化することで、当事者が自分事化しやすい粒度に揃えている。

### CTA は無料 note 一点
バックエンド（マインド構築講座 ¥550,000）／ミドルエンド（有料note ¥500–4,980）には固定ポストから直接導線を引かない。`config/strategy.json:funnel` に従って **SNS → 無料 note → LINE → ミドルエンド → バックエンド** の段階導線を維持する。

### キャリア記述は「元エンジニアの現役コーチ」に圧縮
`docs/author_profile.md` のフル系列「珈琲屋オーナー → エンジニア → エンジニア兼コーチ」のうち、ペルソナ（IT エンジニア）と最も共鳴する 2 要素だけを残して短縮。珈琲屋経歴は bio 側／タイムライン投稿側で補完する想定。
