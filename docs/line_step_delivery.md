# LINE 7日間ステップ配信 本文

最終更新: 2026-05-12

> 本ドキュメントは公式LINE登録者向けの7日間ステップ配信（Day 0〜Day 7）の本文・配信タイミング・運用設計の運用版。LINE登録者を興味段階→理解段階へ進め、無料webセミナーへの参加を促す導線。
>
> ファネル全体設計は `docs/funnel_design.md` §2.3 を参照。セミナー構成は `docs/seminar_slides.md` を参照。個別相談スクリプトは `docs/consultation_script.md` を参照。

## 1. 概要

| 項目 | 内容 |
|---|---|
| 対象 | 公式LINE登録者（短縮URL経由でThreads／note等から登録した方） |
| 配信方法 | LINE Official Account の自動配信（登録日を起点とした日別配信） |
| 必要プラン | LINE 公式アカウント **ライトプラン ¥5,000/月**（無料プランは月200通制限のため不可） |
| 配信総数 | 8通（Day 0〜Day 7） |
| 文字数目安 | 各日 500〜1,500字 |
| 主要KPI | LINE登録→セミナー予約率 50%（`funnel_design.md` 層3-a 目標） |
| 最終ゴール | Day 7 のセミナー申込フォームへの誘導 |

## 2. 設計原則

### 2.1 1日1通・朝配信
登録日（Day 0）を起点に、毎日朝7:00 JST に1通配信。Day 1 は登録翌日。

### 2.2 文字数 500〜1,500字／LINEで読みやすい改行
スマホ画面で読みやすい3〜5行ごとの段落区切りを意識。長文ブロックは避ける。

### 2.3 観察者ポジション
クライアント事例は「クライアントの〇〇さん」形式で語る（`docs/author_profile.md` 準拠）。発信者自身の体験と区別。

### 2.4 心理学・脳科学はメカニズム説明として扱う
出典明記による権威誇示はしない。「自律神経」「扁桃体」などの専門語を1つだけ自然に組み込む。

### 2.5 「学ぶ→変わる」「自己流NG」の前段仕込み
Day 4・Day 5 で `docs/consultation_script.md` §4.3 の仕込み事項（自己投資の必要性／学ぶでは届かない領域／発信者が自己流で躓いた経験）を組み込む。

### 2.6 発信者開示は「一度で解決した成功譚」にしない
Day 5 では「同じ罠を何度もくぐり抜けてきた当事者」のスタンスを保つ（`config/strategy.json` `post_types.personal` 準拠）。

### 2.7 Day 7 のセミナー告知は希少性を演出
「次回開催を逃すと次は◯日後」と明示。申込フォームURLには締切と定員を入れる。

### 2.8 テーマ一貫
「親密になるほど窮屈になっていた関係を、関係を壊しそうな前兆に気づくことで、愛が深まるほど安心できる関係に変える ── 結婚の幸せに踏み込める」を軸に進める。

### 2.9 否定型フック禁止・マイナス語での自己表現禁止
「〜できないあなた」「ダメな自分」など否定型は使わない。`scripts/generate_post.py` の共通ルール準拠。

### Day 2：なぜ親密になるほど窮屈になるのか（脳科学的メカニズム）

**目的**: 興味→理解／構造解明／脳科学根拠1つ

**文字数**: 約 1,000字

**本文**:

```
[お名前]さん、おはようございます。

今日は「なぜ親密になるほど窮屈になるのか」を、
脳の働きの側面からお伝えします。

──

結論から言うと、これは性格の問題でも、相手選びの問題でもなく、
**脳の自然な反応**として起きています。

人間の脳には、自分のテリトリー（心理的な領域）を守る機能があります。
これは生存に必要な機能で、誰でも持っています。

親密度が上がるとき、つまり相手が自分のテリトリーに入ってくるとき、
脳の一部（扁桃体周辺）が「侵入を察知」して反応します。

これが体感としては「窮屈感」「重さ」「冷め」として現れます。

つまり、

・相手が悪いから冷めたわけではない
・自分の愛情が薄いから冷めたわけではない
・脳が「侵入を察知した反応」をしているだけ

これが構造の正体です。

──

ここで重要なのは、この反応は「訓練可能」だということです。

侵入を察知したときに「これは反応であって、相手の問題ではない」とラベル付けできるようになると、過去の自分とは違う選択が取れるようになります。

たとえば、

・冷めたから別れる
　 → 冷めたけど、これは反応だと分かるから留まれる

・窮屈だから距離を取る
　 → 窮屈だけど、これはパターンだと分かるから留まれる

ただし、これは「知識として知っている」だけでは足りません。

実際にその瞬間に「あ、今これが起きている」とリアルタイムで気づける状態を作る必要があります。

このプロセスについては、Day 4以降で詳しくお伝えします。

──

明日は、私がコーチとして実際に観察してきた、
2つのクライアントの事例をお伝えします。

「気づけた人」と「気づけずに壊した人」の対比です。

それでは、また明日。

トーサイ
```

**配信ポイント**:
- 脳科学根拠1つ（扁桃体周辺の侵入察知反応）を自然に組み込む
- 出典明記しない／権威誇示しない

ただ「反応が起きた瞬間にそれを言語化できるかどうか」だとお伝えしました。

──

**学んだはずなのに、自分の問題は解けない。**

──

これは、[お名前]さんの理解力が足りないからではありません。

「学ぶ」と「変わる」が、脳の使い方として、まったく別の能力だからです。

学ぶには、知識のインプットで足ります。
他人を分析するときには、その知識が使えます。

でも、自分が窮屈になった瞬間に「これは私のパターンだ」と気づく能力は、
知識のインプットでは身につきません。

本を100冊読んでも到達しにくい領域です。

ここで、もう1つ重要な事実をお伝えします。

世の中で出回っている恋愛・婚活ノウハウの大半は「テクニック型」です。

「モテるための〇〇」

「結婚相手の選び方」

このプロセスは無料の情報では手に入りません。本でも手に入りません。

なぜなら、本は「他人を分析する素材」を提供してくれますが、
「自分の内側に気づく訓練」までは提供できないからです。

逆に言えば、内側に気づく訓練に投資する価値は、その時間と金額に十分見合います。

──

明日は、私自身の話をします。

私も過去、ここまでお伝えしてきたパターンをずっと繰り返してきた1人です。
それをどう抜けてきたか、現在進行形でどう向き合っているかをお伝えします。

それでは、また明日。

### Day 6：関係を壊しそうな3つの前兆 概要

**目的**: 理解→セミナー期待醸成／3つの前兆の名前のみ提示

**文字数**: 約 900字

**本文**:

```
[お名前]さん、おはようございます。

今日は「関係を壊しそうな3つの前兆」の概要をお伝えします。

私がコーチとして観察してきた中で、
独身ITエンジニア・PM男性の方が関係を壊すパターンには、
大きく3つの型があると見えてきました。

**前兆A：親密反転型**

近づくほど、離れたくなる感覚。

相手が「もっと一緒にいたい」と踏み込んできた瞬間、
自分の中で「窮屈だ」という反応が出る。

これに当てはまる方は、「相手のことは好きだけど、距離を詰められると引いてしまう」という体感を持っています。

──

**前兆B：結婚スパイク型**

結婚が見えた瞬間、急に冷める。

それまで何の問題もなかったのに、結婚の話題が出た瞬間、
「この人で本当にいいのか」という思考が止まらなくなる。

これに当てはまる方は、「結婚」という単語が自分の中で何かをスイッチしているのが特徴です。

──

**前兆C：優しさ消耗型**

大切にするほど、自分が疲弊する。

相手を思いやって自分の本音を抑える選択を続けるうちに、
少しずつ違和感や疲弊が蓄積する。

最終的には「いい人なんだけど、何を考えているか分からない」と振られる、
あるいは静かに撤退する。

──

多くの方は3つのうち1〜2つが主軸になっています。

[お名前]さんは、どの前兆に思い当たりがありますか？

ただし、ここでお伝えしたい重要なことがあります。

**3つの前兆の名前を知るだけでは、何も変わりません。**

知識として知っているのと、自分が窮屈になった瞬間にそれを「これは私の親密反転だ」と言語化できることは、まったく別の能力だからです。

明日、この3つの前兆を実際に解消していくための、私がオリジナルで体系化した『**愛を深め続ける関係性構築戦略**』を体系的にお伝えする90分セミナーのご案内をします。

それでは、また明日。

トーサイ
```

**配信ポイント**:
- 3つの前兆の名前のみを提示（詳細はセミナー）
- ペルソナの自分事化を促す（「どれに思い当たりがありますか？」）
- 「学ぶでは変わらない」の再確認＋セミナー期待醸成

## 4. 配信タイミングとLINEプラン

### 4.1 配信スケジュール

| Day | 配信タイミング | トリガー |
|---|---|---|
| Day 0 | 登録後30秒以内 | 友だち追加イベント |
| Day 1 | Day 0 配信から24時間後・朝7:00 | 自動配信スケジュール |
| Day 2 | Day 1 配信から24時間後・朝7:00 | 同上 |
| Day 3 | Day 2 配信から24時間後・朝7:00 | 同上 |
| Day 4 | Day 3 配信から24時間後・朝7:00 | 同上 |
| Day 5 | Day 4 配信から24時間後・朝7:00 | 同上 |
| Day 6 | Day 5 配信から24時間後・朝7:00 | 同上 |
| Day 7 | Day 6 配信から24時間後・朝7:00 | 同上 |

LINE公式アカウントの **ステップ配信機能** で登録日起点の自動配信を設定。曜日・日付ではなく登録からの相対時間で動かす。

### 4.2 LINEプラン要件

- **ライトプラン ¥5,000/月** 必須
- 配信総数（月間）の概算：登録者100名 × 8通 = 800通／月 → 無料プラン（月200通）では即時超過
- ライトプランの月5,000通枠で 600名/月の登録までは超過課金なしでカバー

### 4.3 7日間後のフォロー配信

Day 7 後の配信は、ステップ配信から **個別運用配信** に切り替え。具体的には:

- セミナー参加者向け：個別相談申込フォーム送付（Day 7+1）
- セミナー欠席者向け：次月開催の案内（Day 7+2 または翌月初日）
- セミナー未予約者向け：1週間後にリマインダー（押し売りにならない範囲で）

## 5. 配信後の運用

### 5.1 開封率・配信完了の確認（週次）

- LINE Official Account Manager で各配信の **配信成功数** と **開封数** を確認
- 開封率が極端に低い日（30%未満）があれば、件名や冒頭文を改善
- 配信失敗（ブロック等）が増えた場合、`ファネル登録ログ` の `status` を `lost` に更新

### 5.2 Day 7 セミナー申込フォーム到達

- 申込時に `ファネル登録ログ` の `seminar_reserved_at` に時刻記録
- `seminar_session_id` に予約セミナー回（YYYY-MM-DD）を記入

### 5.3 月次振り返り（コホート別）

- LINE登録月コホートごとの「Day 7 → セミナー予約率」を確認（目標50%・`funnel_design.md` 層3-a）
- 50% を下回る場合、改善箇所の候補:
  - Day 6 → Day 7 の橋渡し（3つの前兆 → セミナーで詳細）の説得力
  - Day 7 の希少性訴求（開催日明示／個別相談特典）
  - Day 4 の自己投資仕込み（個別相談の¥50,000/h相当の前段）

### 5.4 ステップ配信の更新タイミング

- 3ヶ月運用後、コホート別 CVR を見て本文を改善
- 特定のDayで離脱が多い（次の配信を開封しない）場合、そのDayが原因
- 改善するときは A/B テストではなく一括差し替え（運用負荷の観点）

## 6. 運用時の置換変数一覧

各配信本文中の `[XXX]` は実配信時に運用者が置換する変数：

| 変数 | 内容 | 取得元 |
|---|---|---|
| `[お名前]` | LINE上の表示名 | LINE 自動取得（`{{userName}}` 等の標準変数） |
| `[PDF URL]` | 登録特典PDFのダウンロードURL | Google Drive / Dropbox 等のシェア URL |
| `[セミナー申込フォームURL]` | 月別の Googleフォーム URL | 開催月ごとに発行 |
| `[YYYY/MM/DD（曜日）HH:MM〜HH:MM]` | セミナー開催日時 | 開催月ごとに更新 |
| `[N]` | 個別相談の今月枠数／セミナー定員 | 運用者判断 |

Day 7 は月別に内容が変わるため、月初に運用者が配信内容を更新する。Day 0〜6 は固定で運用可能。
